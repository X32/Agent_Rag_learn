## 一、核心问题（30 分钟）

### 1.1 为什么要用向量数据库？

**先问自己**：第一周的 RAG 系统有个什么问题？

```
第一周的代码：
1. 每次运行 → 重新加载文档 → 重新计算 Embedding → 存入 numpy 数组
2. 用户提问 → 遍历所有向量计算相似度

问题：
- 每次启动都要等很久（重复计算 Embedding，费钱）
- 文档多了之后，遍历所有向量很慢（效率低）
- 无法方便地添加/删除文档（难以维护）
```

**向量数据库解决什么？**

```
┌─────────────────────────────────────────────────────────────┐
│                    向量数据库的价值                          │
├─────────────────────────────────────────────────────────────┤
│  1. 持久化存储：Embedding 算一次，存下来，下次直接用          │
│  2. 快速检索：专门的索引结构，比遍历快 10-100 倍              │
│  3. 增量更新：随时添加新文档，不用重新计算所有向量            │
│  4. 元数据过滤：可以按标签/日期等条件筛选                    │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 FAISS 是什么？

**一句话**：Facebook 开源的向量相似度检索库，速度快、免费、可本地运行。

**核心原理（简化版）**：

```
没有 FAISS：
用户问题 → 遍历 10000 个向量 → 计算 10000 次相似度 → 找最相似的
          ↓ 慢（O(n) 复杂度）

用了 FAISS：
用户问题 → FAISS 索引 → 直接定位到最相似的几个 → 计算几次相似度
          ↑ 快（O(log n) 或 O(1) 复杂度）
```

**类比理解**：

| 方式            | 比喻                     | 速度 |
| --------------- | ------------------------ | ---- |
| 遍历 numpy 数组 | 图书馆没有目录，一本本翻 | 慢   |
| FAISS           | 图书馆有卡片目录，直接查 | 快   |

### 2.3 理解关键代码

**FAISS 索引创建**：

```python
import faiss

# 假设 vectors 是你的 Embedding 向量 (n_samples, n_dimensions)
dimension = vectors.shape[1]  # 获取维度，比如 1536

# 创建索引 - 最简单的方式 (L2 距离)
index = faiss.IndexFlatL2(dimension)

# 或者用余弦相似度 (先归一化)
index = faiss.IndexFlatIP(dimension)  # 内积，归一化后等于余弦

# 添加向量到索引
index.add(vectors)

# 保存到磁盘
faiss.write_index(index, "doc_index.faiss")

# 从磁盘加载
index = faiss.read_index("doc_index.faiss")
```

**检索**：

```python
# 搜索最相似的 3 个文档
k = 3
distances, indices = index.search(query_vector, k)

# distances: 距离列表（越小越相似）
# indices: 对应的文档索引
```

## 三、递归追问（30 分钟）

### 3.1 自己问自己

**问题 1**：FAISS 比直接存 numpy 数组好在哪里？

<details>
<summary>参考答案</summary>

- **速度**：专门的索引结构，检索速度快 10-100 倍
- **功能**：支持增量添加、删除向量
- **存储**：高效的二进制格式，节省空间

</details>

**问题 2**：FAISS 的 IndexFlatL2 和 IndexFlatIP 有什么区别？

<details>
<summary>参考答案</summary>

- **L2**：欧氏距离，距离越小越相似
- **IP**：内积，向量归一化后等于余弦相似度，值越大越相似

</details>

**问题 3**：什么是索引（Index）？

<details>
<summary>参考答案</summary>

索引是一种数据结构，帮助快速找到目标，不用遍历所有数据。
就像书的目录，帮你快速找到章节，不用一页页翻。

</details>

## 四、知识卡片（15 分钟）

### 向量数据库 vs 传统数据库

| 维度     | 传统数据库        | 向量数据库        |
| -------- | ----------------- | ----------------- |
| 存储内容 | 文本/数字/日期    | 向量（一串数字）  |
| 检索方式 | 精确匹配/范围查询 | 相似度搜索        |
| 典型场景 | 订单管理/用户信息 | 语义搜索/推荐系统 |

### 常见向量数据库对比

| 工具     | 类型      | 特点                 | 适用场景          |
| -------- | --------- | -------------------- | ----------------- |
| FAISS    | 本地库    | 免费、快、功能多     | 本地原型/中小规模 |
| Chroma   | 本地/服务 | 简单易用、支持元数据 | 快速开发          |
| Milvus   | 云服务    | 分布式、大规模       | 企业级应用        |
| Pinecone | 云服务    | 托管、免运维         | 不想运维的团队    |

## 六、常见坑

**坑 1**：保存后加载报错

```
原因：FAISS 版本不兼容
解决：尽量用稳定版，不要用太新的 alpha/beta 版
```

**坑 2**：检索结果为空

```
原因：query 向量维度与 index 不一致
解决：print 检查 dimension 是否相同
```

**坑 3**：距离是负数

```
原因：用了 IndexFlatIP（内积），归一化后范围是 [-1, 1]
解决：这是正常的，值越大越相似
```
