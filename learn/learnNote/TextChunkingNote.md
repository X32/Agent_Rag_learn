
# **切分**: 为什么要切分？怎么切的？

### 1. LLM 的输入限制

- 上下文窗口有限 ：LLM 无法一次处理整个文档（几 KB 到 100k tokens 不等）
- 超出限制会被 截断 ，丢失后面的信息

### 2. 检索精度的需要

❌ 不分块：整个文档作为一个向量
   → 检索时只能返回整个文档
   → 包含很多无关内容

✅ 分块后：每个块独立向量化
   → 检索时只返回最相关的几个块
   → 减少噪音，提高准确性

#### 1. 文档类型


```
只要精准答案      → 小块 (200-400)
需要背景信息      → 中等块 (500-800)
```

```
GPT-3.5 (4K)   → 建议 ≤500，留空间给 prompt 和回答
```

## **相似度计算**: cosine similarity 公式是什么？

## Cosine Similarity（余弦相似度）公式

### 1. 数学公式

$$
\text{cosine\_similarity}(A, B) = \frac{A \cdot B}{\|A\| \times \|B\|}
$$

展开写：

$$
\text{cosine\_similarity} = \frac{\sum_{i=1}^{n} A_i \times B_i}{\sqrt{\sum_{i=1}^{n} A_i^2} \times \sqrt{\sum_{i=1}^{n} B_i^2}}
$$

### 2. 公式解读

| 符号          | 含义                                                               |
| ------------- | ------------------------------------------------------------------ |
| $A, B$      | 两个向量（ embedding ）                                            |
| $A \cdot B$ | **点积**：对应元素相乘后求和                                 |
| $\|A\|$     | **向量范数**（长度）：$\sqrt{A_1^2 + A_2^2 + ... + A_n^2}$ |

### 3. 几何理解

```
         B
        /
       /
      /
     /
    /_________ A

 cosine similarity = cos(θ)
                   = 夹角的余弦值
```

- **夹角越小** → cos 值越接近 1 → 越相似
- **夹角越大** → cos 值越接近 0 → 越不相似
- **方向相反** → cos 值为 -1 → 完全不相似

### 4. 值的范围

| 余弦相似度 | 含义         |
| ---------- | ------------ |
| 1.0        | 完全相同方向 |
| 0.8 ~ 0.9  | 非常相似     |
| 0.5 ~ 0.7  | 中等相似     |
| 0.3 ~ 0.5  | 不太相似     |
| 0.0        | 毫无关系     |
| -1.0       | 完全相反     |

### 5. 示例计算

```python
import numpy as np

# 两个简单的 3 维向量
A = [1, 2, 3]
B = [2, 4, 6]

# 手动计算
dot_product = 1*2 + 2*4 + 3*6  # = 2 + 8 + 18 = 28
norm_A = np.sqrt(1**2 + 2**2 + 3**2)  # = 3.74
norm_B = np.sqrt(2**2 + 4**2 + 6**2)  # = 7.48

cos_sim = 28 / (3.74 * 7.48)  # ≈ 1.0

# 使用库
from sklearn.metrics.pairwise import cosine_similarity
cos_sim = cosine_similarity([A], [B])  # = 1.0
```

**结果**：这两个向量方向完全相同（只是长度不同），余弦相似度 = 1.0

### 6. 在 RAG 中的作用

```
用户问题 → 向量化
    ↓
[0.12, -0.34, 0.56, ...]  (384维)
    ↓ 与数据库中每个块计算 cosine similarity
    ↓
返回 top_k 个最相似的块
```

**示例**：

```
查询向量 vs 块1向量 → 0.95 (非常相关)
查询向量 vs 块2向量 → 0.72 (一般相关)
查询向量 vs 块3向量 → 0.31 (不太相关)

返回 top_k=2 → [块1, 块2]
```

### 7. 为什么用余弦相似度？

| 指标                 | 特点                                               |
| -------------------- | -------------------------------------------------- |
| **余弦相似度** | 只看**方向**，不看长度 → 更适合文本语义匹配 |
| 欧几里得距离         | 看绝对距离 → 对长度敏感                           |

文本向量化的目标是捕捉**语义方向**，所以余弦相似度更合适。

## RAG 的 5 个主要步骤

```
┌─────────────────────────────────────────────────────────────┐
│                      RAG 完整流程                            │
└─────────────────────────────────────────────────────────────┘

  1. 📄 文档加载   2. ✂️ 文本分块   3. 🔢 向量化   4. 🔍 检索    5. 💬 生成回答
     (Load)          (Chunk)        (Embed)       (Retrieve)     (Generate)
```

---

### 1️⃣ 文档加载 (Document Loading)

- 把各种格式的文件读进来（PDF、TXT、Markdown...）
- 提取纯文本内容

---

### 2️⃣ 文本分块 (Text Chunking)

- 把长文档切成小块（每块 300-1000 字符）
- 保持语义完整性（尽量按句子切）
- 每个块独立存储和检索

---

### 3️⃣ 向量化 (Embedding)

- 把每个文本块变成一个"向量"（一串数字）
- 意思相近的内容 → 向量也相近
- 用 Embedding 模型（如 SentenceTransformer）

---

### 4️⃣ 检索 (Retrieval)

- 把用户问题也变成向量
- 在向量数据库中找"最相似"的 top_k 个块
- 使用余弦相似度计算

---

### 5️⃣ 生成回答 (Generation)

- 把检索到的相关块+用户问题 → 组成 Prompt
- 发送给 LLM（如 GPT、Claude）
- LLM 生成最终回答
