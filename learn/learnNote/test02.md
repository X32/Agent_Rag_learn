**问题 1：RAG 流程**
请简述 RAG（Retrieval-Augmented Generation）的核心流程，包括从用户提问到生成答案的主要步骤。

答案：读取文档，切分文档，向量化和存储，检索内容，生成回答

用户输入问题进行向量化，从向量数据库检索内容，检索内容和用户输入问题输入大模型生成问题回答

**问题 2：文本切分策略**
为什么在 RAG 中需要对文档进行切分（Chunking）？常见的切分策略有哪些？请列举至少 2 种。

大模型有文本输入长度限制，太长的检索效率低

答案：固定切分，递归切分，按句子切分，按语义切分。

**问题 3：Embedding 的作用**
什么是 Embedding？它在 RAG 系统的哪个环节发挥作用？为什么不能用关键词匹配（如 BM25）完全替代？

答案：将高纬的离散的数据比如文本，图片，等转换成低纬的连续的向量，保留语义结构信息，可计算科学习，保留语义，应用场合NLP，RAG ，等

它在 RAG 系统中文本需要向量化后保存在向量数据库，用户的请求问题也会先向量化后进行检索，关键词匹配有时候并不能检索到语义相同的内容，比如苹果和苹果手机

**问题 4：检索阶段**
在 RAG 的检索阶段，系统如何找到与用户问题最相关的文档片段？请解释"相似度"是如何计算的。

答案 ：余弦相似度，

* **分子** ：点积（对应维度相乘再求和）
* **分母** ：两个向量的长度（模）相乘

**问题 5：实际应用**
假设你要构建一个客服问答 RAG 系统，文档是产品手册。如果发现检索结果经常不相关，可能的原因有哪些？请从文本切分或 Embedding 角度分析至少 2 个可能原因。


### 原因 1：切分粒度不合理

```
场景：产品手册

❌ 错误做法：按固定 500 字切分

原文：「iPhone 15 的电池容量是 3349mAh，
      支持 20W 快充，无线充电功率 15W。
      屏幕尺寸 6.1 英寸，分辨率 2556×1179...」

切分后：
Chunk 1: 「iPhone 15 的电池容量是 3349mAh，支持 20W 快充」
Chunk 2: 「屏幕尺寸 6.1 英寸，分辨率 2556×1179」
```

 **问题** ：用户问「iPhone 15 电池多大」，检索到 Chunk 2 就完全不相关。

```
✅ 正确做法：按语义单元切分

- 电池信息一个 chunk
- 屏幕信息一个 chunk
- 相机信息一个 chunk
```

---

### 原因 2：丢失上下文

```
原文结构：
## iPhone 15
### 电池
- 容量：3349mAh
- 充电：20W 快充

### 屏幕
- 尺寸：6.1 英寸
```

❌  **错误切分** ：只取纯文本

```
Chunk: 「容量：3349mAh，充电：20W 快充」
```

 **问题** ：检索时不知道这个「容量」是哪个产品的。

```
✅ 正确切分：带上父级标题
Chunk: 「iPhone 15 电池：容量 3349mAh，支持 20W 快充」
```

## 从 Embedding 角度

### 原因 3：用通用模型处理专业术语

```
用户问：「支持 PD 快充吗？」
文档：「兼容 USB-PD 3.0 协议」

❌ 通用 Embedding 模型：
「PD」可能被理解成「拼多多」「帕金森」
→ 相似度很低

✅ 领域微调的 Embedding：
知道「PD」= 「Power Delivery」快充协议
→ 相似度高
```

---

### 原因 4：问题与文档的「表达方式」不匹配

```
用户问：「这个手机防水吗？」（口语）

文档：「IP68 级防尘防水」（专业术语）

❌ Embedding 可能认为不相关：
- 用词完全不同
- 一个是疑问句，一个是陈述句

✅ 解决方案：
1. 在文档中加「同义句」
   「IP68 级防尘防水」→ 「这个手机支持防水，等级是 IP68」
   
2. 用 Query 改写
   用户问 → 改写成「IP68 防水等级」再检索
```

## 一个真实案例对比

| 维度                | 错误做法    | 正确做法           |
| ------------------- | ----------- | ------------------ |
| **切分粒度**  | 固定 500 字 | 按语义段落         |
| **上下文**    | 只存纯文本  | 带上标题/产品名    |
| **Embedding** | 通用模型    | 领域微调或加同义句 |
| **Query**     | 直接检索    | 先改写再检索       |

## 快速诊断清单

```
□ 检索结果完全答非所问 → 检查切分是否丢失上下文
□ 检索结果「有点相关但不是答案」→ 检查切分粒度
□ 专业术语检索不到 → Embedding 模型问题
□ 口语问法检索不到 → 表达方式不匹配
```

> **核心原则：让每个 Chunk 独立可读，让 Embedding 能「翻译」用户问题和文档之间的表达差异。**
