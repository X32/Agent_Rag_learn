# RAG 学习计划

> 基于「递归式知识填充」方法，4 周掌握 RAG 核心技术

---

## 学习目标

**项目目标**：搭建一个能回答个人文档问题的知识库助手

**能力目标**：

- 理解 RAG 各组件原理
- 能独立搭建和优化 RAG 系统
- 能分析 RAG 系统的问题并提出解决方案

---

## 时间分配

```
每周 15-20 小时
├── 项目驱动（写代码 + 递归追问）    50%  │ 8-10h

└── 输出反馈（Demo + 博客 + 验证）   20%  │ 3-4h
```

## Week 1：RAG 基础 - 跑起来再说

### 项目任务：最简 RAG 系统

**Day 1-2：让 AI 写代码，跑起来**

提示词：

```
写一个最简单的 RAG 程序：
1. 读取一个 TXT 文件
2. 把文本切分成小块
3. 用 OpenAI API 把文本块转成向量
4. 用户提问时，找到最相似的文本块
5. 把文本块和问题一起发给 LLM，生成答案

要求：代码尽可能简单，每行都要有注释
```

完成后：

- [X] 运行成功
- [X] 逐行理解每一行代码
- [X] 记录不懂的地方

**Day 3-4：递归追问 - 文本切分**

核心问题：

- [X] 为什么要切分？不切分行不行？
- [X] chunk_size 多大合适？为什么？
- [X] chunk_overlap 是什么？为什么需要？
- [X] 有哪些切分策略？（固定长度、句子、语义、递归）

实践：

- [X] 尝试不同的 chunk_size（128, 256, 512, 1024）
- [X] 对比效果差异

**Day 5-6：递归追问 - Embedding**

核心问题：

- [X] Embedding 是什么？为什么能把文字变成数字？
- [X] 向量的维度是什么意思？
- [X] cosine similarity 在算什么？
- [X] OpenAI 的 text-embedding-3-small 和 large 有什么区别？
- [X] 还有哪些 Embedding 模型？（BGE, M3, Voyage）

实践：

- [X] 打印几个文本的向量，观察数值
- [X] 计算几对文本的相似度，感受"语义相似"

**Day 7：输出反馈**

- [X] 写笔记：**"我从零搭建 RAG 学到的 10 件事"**
- [X] 让 AI 检查你的笔记有没有理解错误

## Week 2：向量数据库与检索优化

### 项目任务：升级 RAG - 加上向量数据库

**Day 1-2：向量数据库**

提示词：

```
把我的 RAG 程序升级：
1. 用 FAISS 存储向量（不用每次重新计算）
2. 支持添加新文档
3. 支持持久化存储

解释 FAISS 的工作原理
```

核心问题：

- [X] FAISS 是什么？比直接存 numpy 数组好在哪里？
- [X] 什么是索引？IVF、HNSW 是什么？
- [X] Milvus、Pinecone、Weaviate 有什么区别？
- [X] 什么时候需要用云服务，什么时候本地就够了？

**Day 3-4：检索优化 - Query 处理**

核心问题：

- [ ] 用户的查询可能表达不清，怎么办？
- [ ] Query Rewriting 是什么？
- [ ] HyDE (Hypothetical Document Embedding) 是什么？
- [ ] Multi-Query 是什么？

实践：

- [ ] 实现 Query Rewriting：把用户问题改写得更适合检索
- [ ] 对比改写前后的检索效果

**Day 5-6：检索优化 - Rerank**

提示词：

```
给我的 RAG 加上 Rerank：
1. 先用向量检索召回 Top-20
2. 用 cross-encoder 对 20 个结果重排序
3. 取 Top-5 喂给 LLM

解释 Rerank 的原理
```

核心问题：

- [ ] 为什么向量检索不够，还需要 Rerank？
- [ ] bi-encoder 和 cross-encoder 有什么区别？
- [ ] 常用的 Rerank 模型有哪些？（Cohere, BGE-Reranker）

**Day 7：输出反馈**

- [ ] 对比：基础 RAG vs 加了 Rerank 的 RAG
- [ ] 记录效果提升的数据（如果有测试集）
- [ ] 写笔记：**"RAG 检索优化的 3 个关键技巧"**

## 知识索引（概念地图）

```
RAG 系统
│
├── 文档处理
│   ├── 加载：PDF/Word/HTML/...
│   ├── 切分：固定长度 / 语义 / 递归
│   └── 解析：表格/图片/多模态
│
├── Embedding
│   ├── 模型：OpenAI / BGE / M3
│   ├── 稠密向量 vs 稀疏向量
│   └── 多语言支持
│
├── 存储
│   ├── 向量数据库：FAISS / Milvus / Pinecone
│   ├── 索引类型：Flat / IVF / HNSW
│   └── 元数据过滤
│
├── 检索
│   ├── 基础：向量相似度
│   ├── 进阶：HyDE / Multi-Query
│   ├── 混合：Dense + Sparse (BM25)
│   └── 重排：Rerank
│
├── 生成
│   ├── Prompt 模板
│   ├── Context 组装
│   ├── 引用溯源
│   └── 幻觉检测
│
└── 评估
    ├── 检索评估：召回率 / MRR
    ├── 生成评估：忠实度 / 相关性
    └── 工具：RAGAS / TruLens
```

## 提示词模板

### 学习新概念

```markdown
我想学习 RAG 中的 [概念名称]。请：

1. 用 12 岁小孩能懂的话解释核心思想
2. 给我一个可运行的 Python 代码示例
3. 解释每一行代码在做什么
4. 告诉我这个概念在整个 RAG 系统中的位置
5. 列出相关的 3-5 个重要概念
6. 指出初学者容易误解的地方
```

### 验证理解

```markdown
我对 [概念] 的理解是：

[你的解释...]

请：
1. 指出我理解错误或不完整的地方
2. 用反例挑战我的理解
3. 提出延伸问题测试我的掌握程度
```

### 调试问题

```markdown
我的 RAG 系统遇到了这个问题：

[描述问题]

我的代码：
[贴代码]

请帮我：
1. 分析可能的原因
2. 给出排查步骤
3. 提供解决方案
```
