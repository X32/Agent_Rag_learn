## 核心代码任务

### 任务 1：确认 5 步流程完整

```
1. 读取一个 TXT 文件        → 文档加载
2. 把文本切分成小块          → Chunking
3. 用 API 把文本块转成向量   → Embedding
4. 用户提问时找最相似的块    → Retrieval
5. 把文本块+问题发给 LLM    → Generation
```

### 任务 2：准备测试文件

创建一个简单的 `test.txt`：

```
RAG 是 Retrieval-Augmented Generation 的缩写。
它结合了检索和生成两种技术。
RAG 可以让 AI 回答它训练数据中没有的问题。
向量数据库是 RAG 系统的核心组件。
Embedding 把文本转换成数字向量。
```
